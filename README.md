# 设计原则

## 开闭原则

对扩展开放，对修改关闭

提高软件的可用性和可维护性

## 依赖倒置原则

高层模块不应该依赖低层模块，两者都应该依赖其抽象

针对接口编程，不应该依赖实现编程

## 单一职责原则

不要存在多于一个导致类变更的原因

降低类的复杂度、提高类的可读性

降低变更引起的风险、提高系统的可维护性
